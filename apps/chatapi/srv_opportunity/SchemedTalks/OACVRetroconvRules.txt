jeu de règles complet de génération de CV par défaut :

- objectif : générer en XML une version de CV estimée la plus pertinente pour une offre dont:
  - le lien et la copie du texte sont copiés par moi dans ce prompt,
  - à partir d'un jeu de CV XML appelé "base de CV" uploadé par moi sous la forme d'une archive zip.

- règle "format XML": le format XML des CV de la base de CV et du CV à générer est défini par les règles suivantes:
  - les CV sont conformes au "schéma XSD de CV" donné ci-dessous;
  - il faut respecter systématiquement l’échappement complet des caractères réservés XML dans tous les CV générés;

  - respecter le template suivant :

  <document section-numbering="true" figure-numbering="true" features="RESUME"> 
    <h text-align="justify">...</h> 
    <p/> 
    <section id="Contact" coex="...">...</section> 
    <section id="Objectives" coex="...">...</section> 
    <section id="skills" coex="...">...</section> 
    <section id="Employment" coex="...">...</section> 
    <section id="Education" coex="...">...</section> 
    <section id="Languages" coex="...">...</section> 
    <section id="Interest" coex="...">...</section> 
  </document>

  - les attributs coex doivent contenir le texte du titre de section (<section><h>).
  - dans les sections id="skills" et id="Employment", deux structures sont possibles :
    - liste simple :
      <ul><li><p>...</p></li></ul>
    - sous-section :
      <ul><li><p><strong>...</strong></p><ul><li><p>...</p></li></ul></li></ul>

- règle "format archive base CV": l'archive base CV est un zip contenant un élément <documents> avec plusieurs <document>.

- règle "langue du CV": générer le CV dans la langue de l'offre, si c'est une génréation pour ue offre cible, ou dans la langue du fichier texte source si c'est une conversion d'un CV text  ou (x)html en XML.

- règle "liens dans les CV": si un <a href="..."> est recopié depuis un CV base, conserver le lien.

- règle "surdéfinition": si une règle est surdéfinie dans le prompt, appliquer la version surdéfinie.

- régle "aucun style par API": lorsque le XML est retourné en réponse à l'appel d'un endpoint de l'API openai, il faut renvoyer UNIQUEMENT le contenu XML, strictement sans aucun caractère avant ou après <document>…</document>. Pas de ```xml, pas de markdown, pas de quotes.
